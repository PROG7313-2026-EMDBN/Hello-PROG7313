# Using the Sandbox App to Learn the Skills Needed for Developer Toolkit, MicroTrips, and Digital Detox

## Overview

The Sandbox app is a practice project that helps you build up the exact Android development skills needed for larger applications such as:
- Developer Toolkit
- MicroTrips
- Digital Detox

Repo link: https://github.com/PROG7313-2026-EMDBN/Sandbox

Instead of jumping straight into a full app, you will work through the Sandbox project commit by commit, learning one important concept at a time. Each commit introduces a practical feature that appears again in real-world mobile apps.

This approach allows you to:
- learn incrementally
- understand how features are added over time
- connect individual Android concepts to larger application design
- reuse the same ideas in future projects

## Why We Use Sandbox First
Sandbox gives you a smaller, safer space to practise key skills before applying them in more complete apps.

By working through the commit history, you will learn how to build features such as:
- basic Jetpack Compose screens
- lists and forms
- navigation
- top app bars and drawers
- opening external apps using intents
- loading JSON data
- rendering images
- saving settings with DataStore
- persisting data with RoomDB
- working with multiple tables and relationships
- integrating Firebase Authentication
- separating users and user data
- and more as we go!

These are the same building blocks needed in:

### Developer Toolkit

You will need skills such as:
- structured screens
- navigation between tools
- persistent user preferences
- locally stored data

### MicroTrips

You will need skills such as:
- loading destination data
- rendering lists and images
- storing favourites or saved trips
- navigation across screens
- user-specific saved content

### Digital Detox

You will need skills such as:
- storing sessions and logs
- saving preferences and settings
- handling structured local data
- authenticating users
- separating one user’s data from anothers

## How to Work Through the Sandbox

You should move through the project in the order of the commits. Do not only look at the final version of the app. The value is in understanding how the app evolved.

For each commit:
1. Checkout the commit
2. Run the app
3. Observe what changed
4. Inspect the code carefully
5. Identify the new concept introduced
6. Practice on your own
7. Build your app
